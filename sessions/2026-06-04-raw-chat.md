## 3 error patterns to burn in — these will come back

**1. Python is case-sensitive. Exact.**
```python
Print(height)   # NameError — Python doesn't know "Print"
print(height)   # ✅ lowercase always
```
`print` `Print` `PRINT` are three different things to Python. Only `print` exists.

**2. Variable names must match character-for-character**
```python
you_age = 23           # stored as "you_age"
print(f"{your_age}")   # NameError — "your_age" never existed
print(f"{you_age}")    # ✅ — exact match
```
Python has no idea what you *meant*. It only knows what you *typed*.

**3. `in` is a Python keyword — can't use it as a function name**
```python
you_age = in(input(...))    # SyntaxError — "in" means something else (loops)
you_age = int(input(...))   # ✅
```

## What's installed and how to use it

### 1. ptpython — better REPL (use this instead of `python`)
```
ptpython
```
What it gives you **as you type**:
- Syntax highlighting — `Print` shows in a different colour than `print` so you catch it visually before hitting Enter
- Function signature tooltip — type `print(` and it shows `print(*objects, sep, end, file, flush)` above the cursor
- Tab completion on demand — press `Tab` to see options, not forced on you
- Proper multi-line editing — paste several lines at once

### 2. Error Nudge Hook — fires when you run .py files through Claude Code
When a Python error appears in a Bash run, Claude automatically gets a hint prompt and gives you the error type + one clue — **not the fix**. You find the fix yourself.

### 3. VS Code Pylance — red squiggles before you even run
In any `.py` file:
- Undefined variable → red underline immediately
- Wrong type operation → warning underline
- Hover over any variable → see its inferred type
