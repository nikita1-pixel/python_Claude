# Glossary — plain-language, growing

Add every new term the first time it appears in a session. Keep definitions plain.

| Term | Plain meaning |
|---|---|
| **Python** | A programming language designed to read like English; runs your instructions. |
| **REPL** | Read-Eval-Print Loop — the live scratchpad you get by typing `python3`. Type code, see the result instantly. |
| **script** | A `.py` file holding a saved program you run with `python3 file.py`. |
| **terminal** | The text window where you type commands to the computer (in VS Code: `` Ctrl+` ``). |
| **print()** | A built-in command that shows text on screen. |
| **comment** | A `# note` Python ignores — written for humans. |
| **variable** | A named box that holds a value (`age = 27`). |
| **type** | What kind of value something is — `int`, `float`, `str`, `bool`. |
| **int / float** | Whole number / decimal number. |
| **str (string)** | Text, always inside quotes. |
| **bool** | A True/False value. |
| **f-string** | `f"Hi {name}"` — a string with variables slotted into `{ }`. |
| **operator** | A symbol that does something: `+ - * / // % **`. |
| **modulo (%)** | The remainder after division — `7 % 2` is `1`. |
| **input()** | Asks the user to type something; always returns a string. |
| **type conversion** | Turning one type into another — `int("27")` → `27`. |
| **method** | An action a value can do, called with a dot — `name.upper()`. |
| **PEP 8** | Python's style guide — the conventions that make code look professional. |
