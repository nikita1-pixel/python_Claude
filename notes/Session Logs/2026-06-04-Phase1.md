### Concept 1: Python REPL vs Script Files

> [!info]+ 🔵 Claude explained
> Two ways to run Python:
>
> **REPL** — open with `python` in terminal. You see `>>>`. Type one line, hit Enter, get instant output. Great for experiments — no save, no file, just live feedback.
>
> **Script** — write code in a `.py` file, run with `python filename.py`. For real programs you want to save and repeat.
>
> Rule of thumb: REPL = scratch pad. Script = real work.

### Concept 2: Variables

> [!info]+ 🔵 Claude explained
> A variable is a **named box** that holds a value. You make one with `=` (assignment, not equals-check).
> Python reads the right side first, computes it, then stores it in the name on the left.
>
> Rules for naming:
> - Use lowercase + underscores: `user_name` not `UserName`
> - No spaces, no starting with a number
> - Name should describe what's inside

### Concept 4: Built-in Functions — the Python toolbox

> [!info]+ 🔵 Claude explained
> Python ships with ~70 ready-made tools called **built-in functions**. You call them — you don't build them.
>
> **Pattern:** `function_name(input)` → gives you back a result.
>
> The 7 you need for Phase 1:
>
> | Function | What it does | Key gotcha |
> |---|---|---|
> | `print()` | show output | can take multiple args |
> | `type()` | what type is this? | returns `<class 'x'>` |
> | `len()` | count characters / items | doesn't work on int or float |
> | `input()` | pause, wait for user to type | **always returns a string** |
> | `int()` | convert to integer | fails if input isn't a number |
> | `float()` | convert to decimal | same |
> | `str()` | convert to string | always works |
>
> The most important gotcha: **`input()` always returns a string.** If the user types `22`, you get `"22"` not `22`. You must wrap it: `int(input("Age: "))`.

> [!tip]+ 🟡 Claude showed
> ```python
> # print() — multiple args
> name = "Nikita"
> age = 22
> print("Name:", name, "Age:", age)   # spaces added automatically
>
> # len()
> print(len("Nikita"))        # 6
>
> # input() always returns str — must convert for math
> age = int(input("How old are you? "))
> print(age + 1)              # works now
>
> # Type conversion
> int("22")                   # 22
> float("3.14")               # 3.14
> str(99)                     # "99"

### Concept 5: f-strings

> [!info]+ 🔵 Claude explained
> f-strings are the clean, modern way to put variables inside text.
> Put `f` before the opening quote. Variables (or any expression) go inside `{ }`.
> Every real Python dev uses these — don't use the old `+` concatenation.

> [!tip]+ 🟡 Claude showed
> ```python
> name = "Nikita"
> age  = 22
>
> # Old way (ugly, error-prone):
> print("Hello " + name + ", age " + str(age))
>
> # f-string (clean):
> print(f"Hello {name}, age {age}")
>
> # Expression inside {}:
> print(f"In 10 years: {age + 10}")
> print(f"Name length: {len(name)}")
> ```
